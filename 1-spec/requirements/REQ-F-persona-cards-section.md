# REQ-F-persona-cards-section: New Page Section With Four Persona Cards

**Type**: Functional

**Status**: Draft

**Priority**: Must-have

**Source**: [US-investor-sees-personas](../user-stories/US-investor-sees-personas.md)

**Source stakeholder**: [STK-investor](../stakeholders.md)

## Description

The page must include a new section presenting the four target personas (Lena, Marc, Sophia, Kai) as glass-card elements. Each card must show: archetype name and label, age/profession/city/income range, why they use BAZODIAC (as thinking tool, not belief system), and their monetization role (brand trust / revenue anchor / retention / acquisition). The section must include a persona matrix table mapping roles to subscription tiers and revenue model.

## Acceptance Criteria

- Given the persona section is visible, when an investor reads a persona card, then it contains: archetype name, demographic snapshot (age, profession, city, income), relationship to BAZODIAC (1-2 sentences), monetization role label
- Given all 4 personas, when displayed together, then the four roles (brand standard-setter, revenue anchor, retention engine, acquisition funnel) are clearly differentiated
- Given the section context, when an investor evaluates whether these are real users, then the framing is honest — they are labeled as target personas / proof points, not claimed as existing customers
- Given the design language, when comparing to other sections, then persona cards use the same glass-card styling, gold accents, and typography hierarchy
