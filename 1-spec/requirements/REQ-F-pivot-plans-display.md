# REQ-F-pivot-plans-display: Display Concrete Pivot Scenarios With Asset Preservation Message

**Type**: Functional

**Status**: Draft

**Priority**: Must-have

**Source**: [US-investor-sees-pivot-plans](../user-stories/US-investor-sees-pivot-plans.md)

**Source stakeholder**: [STK-investor](../stakeholders.md)

## Description

The page must display both pivot scenarios with specific trigger conditions and the asset preservation message. Pivot 1: FuFirE as a Service (B2B API licensing for other astrology apps). Pivot 2: Volume Pivot (freemium model at Cosmic Fusion price level, single tier, in-app purchases). Each pivot must be tied to its kill criterion trigger. The section must communicate: "The FuFirE Engine remains valuable in every scenario — only the packaging changes."

## Acceptance Criteria

- Given the pivot section, when an investor reads it, then two pivot scenarios are described: FaaS (B2B API) and Volume Pivot (freemium compression)
- Given each pivot, when checking for trigger conditions, then the specific kill criterion condition is stated (e.g., "Triggered if D30 <7% AND NPS <15% at Day 90")
- Given the asset preservation message, when reading the section, then text communicates that the FuFirE Engine is not a lost asset in any scenario
- Given the overall tone, when evaluating investor perception, then the section reads as "strategic flexibility" not "we might fail"
